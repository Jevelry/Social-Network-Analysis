<h2>Part 3: Discovering Community</h2>

<p><em>Discovering communities</em> in a network, known as community detection/discovery, is a fundamental problem in network science. A network is said to have community structure if the nodes of the network can be easily grouped into sets of nodes such that each set of nodes is densely connected internally. </p>
<!--Descriptions of some of the following items are from Wikipedia at <a href="https://en.wikipedia.org/wiki/Girvan?Newman_algorithm">Girvan?Newman algorithm</a>, adapted for this assignment. -->
<p>The <a href="https://www.pnas.org/content/pnas/99/12/7821.full.pdf" target="_blank"><b>Girvan-Newman algorithm</b></a> detects communities by <em>progressively removing edges</em> from the original network. The connected components of the remaining network are the communities. The Girvan-Newman algorithm focuses on <b>edges</b> that are most likely "<b>between</b>" communities. It adopts the "<b>edge betweenness</b>" centrality introduced in Part 2. The algorithm's steps for community detection are summarized below:</p>

<ol>
    <li>Calculate the edge betweenness of all edges in the network.</li>
    <li>Remove the edge(s) with the highest edge betweenness.</li>
    <li>Recalculate the edge betweenness of all edges affected by the removal.</li>
    <li>Repeat Steps 2 and 3 until no edges remain.</li>
</ol>

<p>After the removal of each edge, we only have to recalculate the betweennesses of those edges that were affected by the removal, which is at most only those in the same component as the removed edge.</p>

<p>The end result of the Girvan-Newman algorithm is a <em>dendrogram</em>. As the Girvan-Newman algorithm runs, the dendrogram is produced from the <em>top down</em> (i.e. the network splits up into different communities with the successive removal of links). The leaves of the dendrogram are individual nodes.</p>

<p> Please see the following simple example, it may answer many of your questions! </p>
<p> <a href="https://www.cse.unsw.edu.au/~cs2521/21T1/assigns/ass2/resources/part-3-example.xlsx">part-3-example.xlsx</a> (MS Excel file) </p>

<p><strong>Note</strong>: If there are multiple edges with the highest edge-betweenness, they should all be removed in the same iteration. Note that, removing these edges may create more than one new connected component. For simplicity, in this assignment, we assume that removing edges during any iteration won't create more than one new component. All the test graphs provided in the Ass2_Testing.zip satisfy this assumption. </p>
